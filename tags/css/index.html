一个垂直居中的方法

这几天看到一些关于垂直居中实现的文章，了解了好多实现垂直居中的方法，在这里把我一个比较喜欢的一个方法记录一下。

根据百度流量研究院的统计，目前(2014年)浏览器的市场份额中IE6+7这两个东西之和已经在10%以下了，而且下降的速度还是挺乐观的，所以，关于垂直居中的方法选择，我的关注点已经不是能否照顾到这两个与众不同的浏览器了，而更关注与对HTML结构的的要求，如是否需要增加额外标签以及HTML是否语义化(semantic)。

因此，不考虑兼容IE6+7，对于大小不固定的多行文字及图片的水平垂直居中，我看到的这些方法中最好的应该是这个:

See the Pen Centering in the Unknown by YiQi (@insideyiqi) on CodePen.

这个方法利用CSS伪元素创建一个”ghost element”，并设置他的高度为100%，然后把它和要垂直居中的内容块都display:inline-block使两者并排及vertical-align:middle，内容块就在垂直居中了。水平居中只需要在外层div设置text-align:center即可。

但这样有一个小问题，仅仅这样不是完美居中的，因为两个inline-block元素之间会有空隙，这在我之前写的各种「间距」问题的总结中有提到过，当时给的终极解决方法是支持IE6-7的，但是这里不需要，因为他们不支持伪元素，所以这里就没必要用那个冗长的CSS，只需要在外层div设置font-size:0然后在内容块设置回来就可以了。

body的background-color问题

在有一次瞎折腾的时候，发现了一个奇怪的问题，在给body标签设置margin值并设置background-color时，background-color竟会扩散到margin部分作用，demo如下：

See the Pen body有margin bg-c扩散 by YiQi (@insideyiqi) on CodePen.

然后当我在给html标签也设置background-color后，才算恢复了”正常”。

See the Pen body有margin html有bgc by YiQi (@insideyiqi) on CodePen.

这勾起了我的好奇心，google之，可是没有找到一个好的解释，倒是stackoverflow上有一些类似的提问，比如这个CSS: background-color only inside the margin，回答者的意思基本都是 要避免这个问题 要不就给html加background-color 要不就不要给body设margin，而是加一个div，给这个div加margin。但貌似找不到解释这个现象的回答，没办法，那就自己翻标准去吧。

W3C给body和html标签的定义是这样的

HTML/Elements/body

The <body> element represents the main content of the document.

HTML/Elements/html

The <html> element represents the root of HTML and XHTML documents. Any subsequent elements are the children of this root element.

好像有点无关痛痒，但我有了个猜想，会不会是这样，当html标签没有设置什么样式的时候，body就成了整个浏览器窗口的全部，因此body的背景色也就被当做了整个页面的背景色。但这也就是猜想，无从求证，我也就先记下了这个问题，暂且作罢。

但过了两天在搜索的过程中找到了一篇张大神的博文，里面谈到了我的这个问题，然后他做出了一个类似的推论

一般情况下，我们css控制的最高结点就是body，例如设置：body{background:#069;}则浏览器界面就是完全的#068的背景色。这里看上去是body标签下的背景色起作用了，我到不这么认为，这里不是body的background起作用，而是body作为一个根结点起作用了，html标签未被激活，body但当类似于根结点的结点，其background背景色被浏览器俘获，浏览器界面背景色为background的背景色，以上是我的推论，这种推论不是我凭空想象出来的，而是有一定的根据的……

博文里对这个推论做了一些实验佐证，但貌似也没有从标准里找到依据，但是这篇博文还介绍了很多关于html和body的相关事情，开了我的眼界，在此推荐一下。

各种「间距」问题的总结

在使用CSS构建页面过程中，有时候会遇到一些莫名的”间距”，让人摸不着头脑，不知道其产生的原理，也不知道该如何消除它，这篇文章就是我所遇到的”间距”问题的总结，随着我遇到的种类增多不断更新。

一、 display 属性值设为 inline-block 的间距

在将一系列的block或inline元素设为display:inline-block后，两个元素会在水平方向产生间距，比如常用来做导航的ul列表的各个li元素在设为display:inline-block后，效果如下

See the Pen inline-block有间隙 by YiQi (@insideyiqi) on CodePen.

关于这个间距是为何产生和如何消除的问题，网上已经有很多的前辈给出了很多详细且深入浅出的博文，在此给出有代表性的两篇，自己也是看了这些文章后才对inline-block有了认识。

  1. inline-block 前世今生
  2. 去除inline-block元素间间距的N种方法

关于这个间距，自己的简单总结是

  1. 这不是bug，反而这是浏览器遵循规范导致的结果(是inline元素本身固有特性)。
  2. 这个间距是由于空白符(white space)导致的，要消灭这个间距，就是要消灭这个空白符，所以有了两个方向的解决方案:

第一是在HTML文档直接消除，比如在上面这个例子中

<ul id="nav">
    <li><a href="#">one</a></li>
    <li><a href="#">two</a></li>
    <li><a href="#">three</a></li>
</ul>

空白符就是</li>和<li>之间的换行符，因此可以这样

<ul id="nav">
    <li><a href="#">one</a></li><li><a href="#">two</a></li><li><a href="#">three</a></li>
</ul>

或是

<ul id="nav">
    <li><a href="#">one</a></li><!--
    --><li><a href="#">two</a></li><!--
    --><li><a href="#">three</a></li>
</ul>

等等，虽然通过直接修改HTML文档可以完美地解决空隙的问题，但是这不符合前端样式控制分离的原则，因此，最好的解决方案还是通过CSS来搞定它。

但是这个问题通过CSS解决却是非常繁杂，当然主要原因还是就在处理各个浏览器的兼容性问题上，上面提到的两篇博文中都介绍了数种方法，就不再赘述，这里就只摘录一个终极全兼容的解决方案吧。

.dib-wrap {
font-size:0;/* 所有浏览器 */
*word-spacing:-1px;/* IE6、7 */
}
.dib-wrap .dib{
font-size: 12px;
letter-spacing: normal;
word-spacing: normal;
vertical-align:top;
}
@media screen and (-webkit-min-device-pixel-ratio:0){
/* firefox 中 letter-spacing 会导致脱离普通流的元素水平位移 */
.dib-wrap{
letter-spacing:-5px;/* Safari 等不支持字体大小为 0 的浏览器, N 根据父级字体调节*/
}
}
.dib {
display: inline-block;
*display:inline;
*zoom:1;
}

其中，在需要设置inline-block元素的父级元素上定义一个.dib-wrap，该元素自身定义为.dib。
